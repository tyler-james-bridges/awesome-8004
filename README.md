## What is 8004?

ERC-8004 is a trustless extension of Google's A2A Protocol for on-chain agents. Learn more by exploring the resources below.

### First Reads

Essential documentation to understand the protocol:

- [**EIP-8004 Specification**](https://eips.ethereum.org/EIPS/eip-8004) - The official Ethereum Improvement Proposal
- [**A2A Protocol Specification**](https://a2a-protocol.org/latest/specification/) - Google's Agent-to-Agent protocol specification
- [**ERC-8004 Contracts**](https://github.com/erc-8004/erc-8004-contracts) - Official smart contract implementations

## Getting Started

Guides and resources to help you start building with ERC-8004.

### Written Guides

- [**Agent0 Quick Start**](https://sdk.ag0.xyz/3-examples/3-1-quick-start/) - Quick start guide for Agent0 SDK
- [**Intensive Learn Trustless Agents**](https://intensivecolearn.ing/en/programs/trustless-agents) - Comprehensive learning program for ERC-8004

### Implementation Examples

- [**ACK (Agent Consensus Kudos)**](https://github.com/tyler-james-bridges/ack-protocol) - Peer-driven reputation layer for AI agents on Abstract. Onchain kudos, MCP/A2A/OASF endpoints, x402 payments, and an npm SDK ([@ack-onchain/sdk](https://www.npmjs.com/package/@ack-onchain/sdk))
- [**Chaos Chain**](https://github.com/ChaosChain/trustless-agents-erc-ri) - Trustless agents implementation
- [**Phala**](https://github.com/Phala-Network/erc-8004-tee-agent) - TEE agent implementation
- [**Zpaynow**](https://github.com/zpaynow/8004) - Rust implementation
